Stock Market Data Pipeline & AI Reporting System

This is my main personal project. I wanted to build a system for monitoring stock information and spotting trends. It ingests stock data from multiple sources, cleans and enriches it, stores everything in MongoDB, and sends daily market summaries by email using LLMs. The whole thing is orchestrated with Airflow, containerized with Docker, and runs on a $12/mo VPS.

Problem

Market data is scattered across sources, formats are inconsistent, and nothing talks to anything else. I wanted a system that could pull it all together into a clean daily picture — something that could eventually support portfolio tracking, reporting, or customer-facing features.

What It Does

  • Ingests raw market data from multiple sources
  • Applies cleaning and normalization rules
  • Merges records across sources and markets
  • Computes enrichment metrics
  • Stores structured outputs in MongoDB
  • Generates daily market summaries using LLMs and sends them via email
  • Includes an NL query agent that answers stock-related questions by querying the pipeline's enriched tables

My Responsibilities

  • Designed the architecture and data flow
  • Built ingestion, transformation, and enrichment logic in Python
  • Scheduled and monitored jobs with Airflow
  • Managed MongoDB schema, collections, and retention logic
  • Containerized services with Docker
  • Deployed and operated the system on a VPS
  • Integrated OpenAI API for report generation
  • Built an LLM-powered agent that interprets natural-language questions and queries the pipeline's data tables to produce answers

Tech Stack

Python SQL Airflow Docker MongoDB Linux / VPS GitHub OpenAI API LLM Agents

Engineering Challenges

  • Normalizing inconsistent source formats
  • Merging records from different exchanges and sources
  • Handling ticker and company-name inconsistencies
  • Designing retention and TTL logic
  • Keeping the system reliable on limited infrastructure
  • Separating configuration, prompts, and business logic cleanly
  • Grounding agent responses in actual pipeline data to avoid hallucination
